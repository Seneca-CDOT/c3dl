C3DLPicking Back-Bearing Test

This pages tests to make sure that picking does not detect points behind the camera.
An issue was identified where an object that extended behind the camera could have
the portion behind the camera show up as picked. This occured because the ray used
for picking did not determine the direction from the source to the object it hit.
In short, a mirror image of the ray was being fired in the opposite direction and
hitting things behind the camera. Clicking slightly above the end of the road
that projects into the scene (and behind the camera) caused it to show up as picked
(and start spinning). This should no longer happen.
